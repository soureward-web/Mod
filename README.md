# نظام إيجاد العملاء المحتملين وإدارة المبيعات

تطبيق Python متكامل لإيجاد عملاء محتملين قريبين منك (Google Maps)، استيراد عملاء من CSV/Excel،
إدارة منتجاتك المادية، إنشاء عروض أسعار PDF تلقائية، وتتبع حالة كل عميل عبر لوحة تحكم Streamlit.

---

## 🗂️ هيكل المشروع

```
leadgen_app/
│
├── app.py               # لوحة التحكم الرئيسية (Streamlit) - نقطة الدخول
├── database.py           # نماذج قاعدة البيانات (SQLite + SQLAlchemy)
├── scraper_maps.py        # جلب عملاء محتملين من Google Places API
├── csv_importer.py        # استيراد عملاء من ملفات CSV/Excel
├── quotes.py             # إنشاء عروض أسعار وتصدير PDF/Excel
├── requirements.txt       # المكتبات المطلوبة
├── data/
│   ├── leadgen.db         # قاعدة البيانات (تُنشأ تلقائيًا عند أول تشغيل)
│   └── example_clients.csv # ملف تجريبي للاستيراد
└── exports/              # مكان حفظ ملفات PDF/Excel المُصدَّرة
```

## ⚙️ التشغيل على Windows

1. **تثبيت Python** (إن لم يكن مثبتًا): حمّله من python.org، وتأكد من تفعيل خيار
   "Add Python to PATH" أثناء التثبيت.

2. **فتح موجه الأوامر (CMD أو PowerShell)** داخل مجلد المشروع:
   ```
   cd المسار\إلى\leadgen_app
   ```

3. **إنشاء بيئة افتراضية (موصى به):**
   ```
   python -m venv venv
   venv\Scripts\activate
   ```

4. **تثبيت المكتبات:**
   ```
   pip install -r requirements.txt
   ```

5. **إعداد مفتاح Google Maps API** (اختياري لكن ضروري لميزة البحث عن عملاء):
   ```
   set GOOGLE_MAPS_API_KEY=مفتاحك_هنا
   ```
   (لجعله دائمًا: أضفه إلى "Environment Variables" في إعدادات Windows)

6. **تشغيل التطبيق:**
   ```
   streamlit run app.py
   ```
   سيفتح المتصفح تلقائيًا على `http://localhost:8501`

## 🔑 كيفية الحصول على إحداثيات منطقتك (Latitude/Longitude)

افتح Google Maps على متصفحك → ابحث عن منطقتك → انقر بزر الفأرة الأيمن على الموقع →
سيظهر لك رقمان (خط العرض، خط الطول) مثل: `33.5731, -7.5898` → انسخهما في التطبيق.

## 📈 الخطة التنفيذية المقترحة (3 مراحل)

### المرحلة 1 — MVP (أسبوع إلى أسبوعين)
الهدف: نظام يعمل محليًا ويحل المشكلتين الأساسيتين بأبسط شكل ممكن.

- [x] قاعدة بيانات SQLite (عملاء، منتجات، عروض أسعار) — جاهزة في `database.py`
- [x] البحث عن عملاء عبر Google Places API حول موقعك — جاهز في `scraper_maps.py`
- [x] استيراد عملاء من CSV/Excel — جاهز في `csv_importer.py`
- [x] لوحة تحكم Streamlit بسيطة: عرض العملاء، تغيير الحالة، إحصائيات أساسية
- [x] إنشاء عرض سعر وتصديره PDF
- [x] تصدير قائمة عملاء إلى Excel

**ما عليك فعله الآن:** شغّل التطبيق، جرّب البحث بمنطقتك، أضف منتجاتك، وأنشئ أول عرض سعر.

### المرحلة 2 — تطوير (3-4 أسابيع)
الهدف: تحسين جودة البيانات وتوسيع مصادر العملاء وتسهيل المتابعة اليومية.

- **مصادر جديدة:** ربط LinkedIn (عبر أدوات مثل PhantomBuster أو Apify - لأن LinkedIn يمنع
  السكرابينج المباشر بشدة وتحظر الحسابات بسرعة)، إضافة Google Search (Custom Search API)
  لجلب مواقع الشركات المحلية.
- **بحث ذكي متعدد المناطق:** بدل نقطة واحدة، أدخل عدة أحياء/مدن دفعة واحدة (شبكة نقاط تلقائية
  لتغطية مدينة كاملة بدل نطاق واحد محدود بـ Google لـ 60 نتيجة).
- **تذكيرات المتابعة:** عمود "تاريخ آخر تواصل" + تنبيه تلقائي "لم يتم التواصل منذ X أيام".
- **بحث وفلترة متقدمة** في قائمة العملاء (حسب الفئة، الحالة، المدينة، التقييم).
- **سجل تفاعلات (Activity Log):** كل مكالمة/زيارة تُسجَّل بتاريخها، بدل حقل "ملاحظات" وحيد.
- **قوالب عروض أسعار متعددة** (شعار الشركة، ألوان، شروط الدفع) بدل قالب واحد ثابت.
- **نسخ احتياطي تلقائي** لقاعدة البيانات (نسخ ملف `leadgen.db` يوميًا).

### المرحلة 3 — إطلاق وتوسع (شهر فأكثر)
الهدف: تحويل الأداة الشخصية إلى نظام قابل للاستخدام من فريق كامل، ومتاح من أي مكان.

- **الترحيل من SQLite إلى PostgreSQL:** بما أن الكود مبني على SQLAlchemy، التغيير يكون فقط
  في سطر `create_engine(...)` (من `sqlite:///...` إلى `postgresql://user:pass@host/db`)
  بدون تعديل باقي الكود تقريبًا.
- **استضافة سحابية:** نشر التطبيق على Streamlit Community Cloud (مجاني للبداية) أو
  Railway/Render لقاعدة البيانات + التطبيق معًا.
- **تعدد المستخدمين + الصلاحيات:** إضافة تسجيل دخول (streamlit-authenticator) بحيث كل مندوب
  مبيعات يرى عملاءه فقط، والمدير يرى الكل.
- **تكامل مع WhatsApp Business API** لإرسال عروض الأسعار مباشرة للعميل من داخل التطبيق.
- **تقارير تحليلية:** معدل التحويل من "جديد" إلى "مشترٍ"، أفضل المصادر أداءً، متوسط قيمة الصفقة.
- **نظام تنبيهات بالبريد الإلكتروني** عند وصول عميل جديد أو تغيّر حالة صفقة مهمة.

## 🧰 الأدوات المستخدمة ولماذا

| الأداة | الاستخدام | لماذا هذا الاختيار |
|---|---|---|
| **SQLite + SQLAlchemy** | قاعدة البيانات | خفيفة، بدون تثبيت سيرفر منفصل، وSQLAlchemy تجعل الترحيل لـ PostgreSQL لاحقًا سطرًا واحدًا فقط |
| **Streamlit** | واجهة المستخدم | أسرع طريقة لبناء لوحة تحكم تفاعلية بـ Python فقط بدون HTML/JS، ومناسبة جدًا لغير المبرمجين المتقدمين |
| **Google Places API** | جلب العملاء المحتملين | رسمي وموثوق (بعكس سكرابينج Selenium المباشر الذي يخالف شروط جوجل ويُعرّض الحساب للحظر)، ودقيق بالهاتف والتقييم والموقع |
| **Pandas** | معالجة الجداول والاستيراد/التصدير | معيار الصناعة لمعالجة البيانات الجدولية في Python |
| **fpdf2** | توليد عروض الأسعار PDF | مكتبة خفيفة بدون تعقيد، كافية تمامًا لمستندات عروض الأسعار |
| **openpyxl** | تصدير Excel | يعمل تلقائيًا خلف `pandas.to_excel()` |

## ⚠️ ملاحظة مهمة حول جلب البيانات من LinkedIn

سكرابينج LinkedIn مباشرة (Selenium/BeautifulSoup) يخالف شروط الاستخدام الخاصة بالمنصة
ويعرّض حسابك الشخصي للحظر الدائم. الطريقة الآمنة والموصى بها في المرحلة 2 هي استخدام
أدوات وسيطة متخصصة ومرخّصة (PhantomBuster، Apify، أو LinkedIn Sales Navigator API الرسمي
إن كان متاحًا لحسابك) بدل بناء سكرابر خاص لهذه المنصة تحديدًا.

## ☁️ النشر السحابي (حتى يعمل من الهاتف من أي مكان)

هذا يحوّل التطبيق من "يعمل فقط عندما يكون كمبيوترك مفتوحًا" إلى رابط دائم مثل
`yourapp.streamlit.app` تفتحه من هاتفك في أي وقت، بدون تشغيل أي شيء على جهازك.

### الخطوة 1 — إنشاء قاعدة بيانات PostgreSQL مجانية (Supabase)

1. اذهب إلى https://supabase.com وأنشئ حسابًا مجانيًا
2. أنشئ مشروعًا جديدًا (اختر منطقة قريبة منك جغرافيًا لسرعة أفضل)
3. من "Project Settings -> Database"، انسخ "Connection string" (وضع URI)
   شكله: `postgresql://postgres:[password]@[host]:5432/postgres`
4. احتفظ بهذا الرابط، ستحتاجه في الخطوة 3

### الخطوة 2 — رفع المشروع على GitHub

1. أنشئ حسابًا على https://github.com إن لم يكن لديك
2. أنشئ مستودع (Repository) جديد باسم مثلاً `leadgen-app`
3. من داخل مجلد المشروع على جهازك:
   ```
   git init
   git add .
   git commit -m "أول نسخة من التطبيق"
   git branch -M main
   git remote add origin https://github.com/اسم_المستخدم/leadgen-app.git
   git push -u origin main
   ```
   (ملف `.gitignore` المرفق يمنع رفع قاعدة البيانات المحلية والأسرار بالخطأ)

### الخطوة 3 — النشر على Streamlit Community Cloud

1. اذهب إلى https://share.streamlit.io وسجّل الدخول بحساب GitHub
2. اضغط "New app" واختر المستودع `leadgen-app` وملف `app.py` كنقطة دخول
3. قبل الضغط على Deploy، افتح "Advanced settings -> Secrets" والصق:
   ```
   GOOGLE_MAPS_API_KEY = "مفتاحك"
   DATABASE_URL = "postgresql://postgres:[password]@[host]:5432/postgres"
   ```
   (نفس محتوى `.streamlit/secrets.toml.example` لكن بقيمك الحقيقية)
4. اضغط Deploy — بعد دقيقة أو اثنتين يصير لديك رابط دائم مثل:
   `https://leadgen-app-xxxx.streamlit.app`

### الخطوة 4 — فتحه من الهاتف

- افتح الرابط من متصفح الهاتف (Chrome أو Safari)
- اضغط قائمة المتصفح -> "إضافة إلى الشاشة الرئيسية" فتصير له أيقونة تفتح مباشرة كأنه تطبيق

### ملاحظات مهمة

- **كل تعديل ترفعه على GitHub (`git push`) يُحدَّث تلقائيًا على الرابط المنشور** خلال ثوانٍ.
- الكود لم يتغيّر أبدًا للتبديل بين SQLite (محليًا) و PostgreSQL (سحابيًا) — فقط وجود
  `DATABASE_URL` في الأسرار هو ما يحدد أيهما يُستخدم (راجع `database.py`).
- خطة Supabase و Streamlit Cloud المجانيتان تكفيان بسهولة لحجم مئات العملاء شهريًا.
